SLIDE 06L21101NLP 演進

NLP 核心演進：Word2Vec → BERT → GPT

Word2Vec（2013）
  • 淺層雙層神經網路，CBOW / Skip-gram
  • 靜態詞向量：每個詞固定一個向量
  • 缺點：無法處理一詞多義
  • 代表作：Mikolov et al., 2013
BERT（2018）
  • Transformer Encoder 雙向架構
  • MLM（遮罩語言模型）預訓練
  • 動態上下文感知向量
  • 適合：分類、問答、NER
GPT 系列（2018–現在）
GPT-1/2/3（解碼器-only）
  • 單向自迴歸生成
  • Next Token Prediction
  • few-shot / zero-shot 能力
RLHF + InstructGPT
  • 人類回饋強化學習
  • 對齊人類意圖
  • GPT-3.5 / GPT-4 基礎
GPT-4o / o1 / o3
  • 多模態原生整合
  • CoT / 推理能力
  • Tool use / Agent
必考：BERT 雙向 / GPT 單向的差異 必考：MLM vs Next Token Prediction 必考：Transformer Encoder vs Decoder

SLIDE 09L21104多模態核心

CLIP：多模態跨世代學習的基石

🔑 CLIP 三大核心能力
  • 零樣本分類：无需训练即可识别新类别
  • 跨模態對比學習：圖像-文字配對對比 loss
  • 文生圖核心：Stable Diffusion 的 CLIP Text Encoder
🧠 架構原理
  • 雙塔架構：Vision Encoder + Text Encoder
  • 對比學習：最大化匹配配對 cosine similarity
  • 4億圖文對預訓練（WebImageText）
  • 訓練目標：對稱交叉熵 loss
📌 為何 CLIP 是多模態關鍵？
  • 打破「每種任務需專門數據」的限制
  • 文本作為橋梁：任意視覺概念可被文本描述
  • 支撐：BLIP, LLaVA, GPT-4V 等多模態模型
  • Prompt Engineering 在視覺領域的延伸
🎨 Stable Diffusion 離不開 CLIP
  • CLIP Text Encoder：將文字 prompt 轉為 embedding
  • U-Net：在潛空間預測噪聲殘差
  • VAE 解碼器：潛空間 → 像素空間
  • CFG（Classifier-Free Guidance）提升質量

📐 PSI（Population Stability Index）
PSI = Σ (A_i − E_i) × ln(A_i/E_i)
PSI < 0.1：無顯著漂移　PSI 0.1~0.2：輕微漂移　PSI > 0.2：嚴重，需重訓

SLIDE 13L21201可行性 + RAI

五大可行性評估 × RAI 六原則

🔧 技術可行性
  • 現有技術能否實現？
  • 模型成熟度 / 準確率要求
  • 算力基礎設施是否足夠
📊 數據可行性
  • 數據量是否足夠訓練模型？

  • 產業監管要求
  • 公平性與非歧視規範
🏢 組織可行性
  • 高層支持與文化接受度
  • 跨部門協作機制
  • 員工培訓與轉型計畫
💵 財務可行性
  • TCO（總擁有成本）估算
  • ROI / 投資回收期
  • Opportunity Cost

  • 公平性：減少偏見與歧視
  • 可靠性：輸出穩定安全
  • 隱私：資料保護機制
  • 包容性：多元利害關係人
  • 透明性：可解釋性與可追溯
  • 問責：責任歸屬機制

SLIDE 20L22101統計基礎

統計基礎：集中趨勢 Mean vs Median vs Mode

μ / x̄
算術平均數 Mean
x̄ = (Σxᵢ) / n
  • 所有數值加總÷個數
  • ✅ 最適合對稱分布
  • ⚠️ 受極端值（離群值）影響大
Med
中位數 Median
排序後正中間的值
（偶數個取兩中點平均）
  • ✅ 不受離群值影響
  • ✅ 偏態分布首選指標
  • 離群值多的薪資/房價首選
Mo
眾數 Mode
出現頻率最高的值
（可有多個眾數）
  • 適合類別型資料（離散）
  • ✅ 類別：性別/城市/產品類型
  • 可用於發現主要類別/高峰值
💡 考試技巧：「有離群值」→ 選 Median。「類別資料」→ 選 Mode。「對稱分布無離群值」→ Mean 資訊最完整
SLIDE 21L22101變異程度

統計基礎：變異程度四指標

📏
全距 Range
R = Max − Min
簡單但極度受離群值影響

⚠️ 最不穩健
📦
四分位距 IQR
IQR = Q3 − Q1
不受兩端離群值影響

✅ 穩健指標
σ²
變異數 Variance
σ² = Σ(xᵢ−μ)²/n
每個偏差平方後的平均

單位是原始單位的平方
σ
標準差 SD
σ = √σ²
標準化的變異程度

⭐ 考試最常用
🔑 變異係數 CV（Coef. of Variation）： CV = σ/μ × 100% — 用於比較不同單位/量級資料的變異程度。CV > 30% 表示高變異，CV < 15% 表示低變異

假設檢定五大步驟（每年必考）

1️⃣
建立假設
H₀：μ = μ₀（原假設）
H₁：μ ≠ μ₀（對立假設）
H₀ 通常是「無差異/無效果」
2️⃣
選擇顯著水準 α
α = 0.05（最常見）
α = 0.01（嚴格）
第一類錯誤容忍上限
3️⃣
計算統計量
Z = (x̄−μ₀)/(σ/√n)
t = (x̄−μ₀)/(s/√n)
Z用於大樣本(n≥30)，t用於小樣本
4️⃣
查 p-value
p = P(觀察到這麼極端的數據|H₀為真)
越小越有證據拒絕H₀
5️⃣
決策與結論
p ≤ α → 拒絕 H₀
p > α → 無法拒絕 H₀
⚠️ 永遠不說「接受 H₀」，只說「沒有足夠證據拒絕 H₀」
SLIDE 23L22102兩類錯誤

兩類錯誤：α 型（誤報）vs β 型（漏報）

❌ 第一類錯誤（Type I / α）
H₀ 為真
✗
錯誤拒絕
  • 「誤報」：把沒效果的當成有效果
  • 例：藥物無效但說有效
  • 設定顯著水準 α 控制此錯誤
  • α↓ → 更嚴格 → β↑（兩者 trade-off）
❌ 第二類錯誤（Type II / β）
H₁ 為真
✗
錯誤接受
  • 「漏報」：把有效果的當成沒效果
  • 例：藥物有效但說無效
  • 1 − β = 統計檢定力（Power）
  • Power ↑ → 增加樣本量 / 提升效應值
⚖️ Power（統計檢定力）= 1 − β
↑ 樣本量 n
↓ 標準誤 → Power ↑
↑ 效應值（Effect Size）
真實差異越大越容易偵測
↓ 變異性
數據越集中差異越明顯

SLIDE 31L23101🔥 必考

ML 數學基礎：貝氏定理 + ROC/AUC 曲線

📐 貝氏定理（L23101）
P(A|B) = P(B|A) × P(A) / P(B)

後驗 = (似然 × 先驗) / 邊際機率
  • P(A)：先驗概率（Prior），觀察前的信念
  • P(B|A)：似然（Likelihood），A 成立時 B 的概率
  • P(A|B)：後驗概率（Posterior），觀察 B 後更新的信念
  • 典型應用：垃圾郵件分類、醫療診斷、文件分類
📊 樸素貝氏（Naive Bayes）
  • 假設：特徵之間條件獨立（常現實不符但有效）
  • 常用：垃圾郵件分類、新聞分類
  • 優點：訓練/預測快速，對小樣本友好
  • 缺點：特徵依賴性強，類別不平衡敏感
📈 ROC / AUC 曲線（L23102）
  • X軸：FPR（偽陽率）= FP / (FP+TN)
  • Y軸：TPR（真陽率）= TP / (TP+FN)
  • AUC = ROC 曲線下面積，0.5（隨機）~ 1.0（完美）
  • AUC 越大 → 模型區分正負類能力越強
必考：何時用 AUC 而非 Accuracy？
不平衡數據時！ Accuracy 會被多數類主導，AUC 考慮所有閾值下的表現
🔑 混淆矩陣速記
TP（真陽）實際正、預測正 ✅
FP（偽陽）實際負、預測正 ❌
TN（真陰）實際負、預測負 ✅
FN（偽陰）實際正、預測負 ❌
SLIDE 32L23103線性代數

ML 數學：PCA / SVD / L1/L2 正則化

📐 PCA（主成分分析）
  • 目標：找出最大變異方向（主成分）
  • 步驟：計算共變異數矩陣 → 求特徵值/向量 → 投影到 Top-k 維度
  • 幾何意義：在保留最多資訊下降低維度
  • 公式：最大化 Var(wᵀX)，約束 ||w|| = 1
  • 優點：去噪、壓縮、加速訓練
  • 缺點：可解釋性下降，非線性關係捕捉不到
🔢 SVD（奇異值分解）
  • A = U × Σ × Vᵀ（U, V 正交，Σ 對角奇異值）
  • PCA 與 SVD 的關係：PCA 是 SVD 的另一種視角
  • 用於：推薦系統（矩陣分解）、降維、壓縮
  • 優點：數值穩定，適用任何矩陣
  • 在推薦系統：User-Item 矩陣分解找隱因子
⚖️ L1（Lasso）vs L2（Ridge）正則化
  • L1（Lasso）：||w||₁ = Σ|wᵢ| → 使部分權重為零（特徵選擇）
  • L2（Ridge）：||w||₂² = Σwᵢ² → 收縮權重但不歸零
  • Elastic Net = L1 + L2 結合
Loss_L1 = Loss₀ + λ·Σ|wᵢ|
Loss_L2 = Loss₀ + λ·Σwᵢ²
必考：L1 稀疏解（特徵選擇）/ L2 平滑解 必考：PCA 找最大變異方向，SVD 任意矩陣分解
SLIDE 33L23104數值優化

ML 數學：梯度下降 GD / SGD / Mini-batch / Adam

📐 梯度下降核心公式
w := w − α · ∇L(w)

α = learning rate（學習率）
∇L(w) = 損失函數梯度
  • 沿梯度反方向更新，收斂到局部/全局最小值
  • 學習率太大震盪發散，太小收斂慢
⚡ 四種變體對比
方法每步樣本速度收斂
GD（批量）全部 n 筆慢穩定
SGD1 筆快震盪
Mini-batch32~256筆中等平衡
Adam自適應快最快
Adam = Adaptive + Momentum：結合 RMSProp 與動量法，自適應學習率
🧠 Momentum（動量法）
v := β·v + (1−β)·∇L(w)
w := w − α·v
像滾球下山：累積歷史梯度方向，加速收斂、減少震盪。β 通常設 0.9

  • 非線性嵌入，高維→2D/3D

🚀 Boosting
Sequential Error Correction
  • 串聯：每個模型修正前一個的錯誤
  • 給錯誤樣本更高權重
  • 減少 Bias（偏差）
  • 串聯訓練，有過擬合風險
⭐ GBDT → XGBoost → LightGBM → CatBoost
XGBoost：正則化 + 二階導數優化
LightGBM：Histogram + GOSS 加速
🔗 Stacking
Stacked Generalization
  • 兩層：Base Models → Meta Model
  • Base：LR、SVM、Tree、KNN 等多樣模型
  • Meta：用 Base 預測結果當輸入，訓練最終預測器
  • 通常用 Cross-Validation 產生 Base 預測
  • 結合多視角，減少單一模型盲點

架構擅長資料類型核心機制代表作必考重點

SLIDE 39L23401模型訓練

模型訓練與評估：Train/Val/Test 劃分 + K 折交叉驗證

📊 資料劃分比例
  • 小數據（<1萬筆）：Train 80% / Val 10% / Test 10%
  • 大數據（>100萬筆）：Train 98% / Val 1% / Test 1%
  • 時序資料：嚴禁隨機劃分，須用時間先後順序
  • 分層抽樣（Stratified）：保持類別比例一致
⚠️ Test Set 只用一次！最終評估，訓練過程不可接觸
🔄 K 折交叉驗證（CV）
  • 將資料分 K 份，輪流取 1 份當 Val，K-1 份當 Train
  • K=5 或 K=10 最常見
  • 比單次劃分更穩定，減少運氣成分
  • Leave-One-Out CV（LOOCV）：K=n，適合小樣本
平均分數 = (score₁ + score₂ + ... + score_K) / K
📈 分類指標全集
Accuracy(TP+TN)/(TP+TN+FP+FN)
PrecisionTP/(TP+FP)　→ 誤殺代價高時
Recall / TPRTP/(TP+FN)　→ 漏檢代價高時
F1 Score2·P·R/(P+R)　→ Precision/Recall 平衡
SpecificityTN/(TN+FP)
📉 回歸指標全集
MSE平均平方誤差，對大誤差懲罰更大
RMSE√MSE，與 Y 同單位
MAE平均絕對誤差，對離群值不敏感
R²決定係數，0~1，越接近1越好
MAPE平均絕對百分比誤差，相對誤差
SLIDE 40L23402🔥 必考

正則化 + 過擬合解決方案

📐 L1 正則化（Lasso）
  • Loss = Loss₀ + λ·Σ|wᵢ|
  • 可使部分權重 exactly 為零 → 特徵選擇
  • 適合：高維稀疏資料（如文本）
  • 缺點：只選一個特徵時可能選錯
📐 L2 正則化（Ridge）
  • Loss = Loss₀ + λ·Σwᵢ²
  • 收縮所有權重但不歸零 → 平滑解
  • 適合：高度共線性特徵
  • Deep Learning 中稱 Weight Decay
🛑 Dropout
  • 訓練時隨機斷開一定比例神經元
  • 防止神經元共同適應（Co-adaptation）
  • 推理時所有神經元啟用，權重縮放
  • 常見：Dropout rate 0.2 ~ 0.5
⏹️ Early Stopping
  • 監控 Val Loss，連續 N 個 epoch 無改善則停止
  • 防止 Train Loss 繼續下降但 Val Loss 上升
  • 最簡單有效的防過擬合手段
📈 Data Augmentation
  • 圖像：旋轉、翻轉、裁剪、色彩變換、MixUp
  • 文本：同義詞替換、Back Translation、Random Insertion
  • 音訊：時間拉伸、音調變換、加噪聲
  • 增加訓練樣本多樣性
🧬 Batch Normalization
  • 每批次內標準化神經元輸入
  • 使訓練更穩定，收斂更快
  • 自帶轻微正則化效果
  • 公式：γ·(x−μ)/σ + β
SLIDE 41L23403偏差-變異數

偏差-變異數權衡 + 超參數搜尋策略

📐 偏差-變異數分解
Total Error = Bias² + Variance + Irreducible Noise

Bias²：模型假設偏離真實關係（欠擬合）
Variance：對訓練資料變化的敏感性（過擬合）
  • 高偏差：模型太簡單，Train/Val 都差 → 複雜化模型
  • 高變異：Train 好、Val 差 → 增加數據/正則化/簡化
  • 目標：兩者同時降低（通常有 trade-off）
🎯 超參數搜尋策略
  • 網格搜尋（Grid Search）：窮舉所有組合，暴力但全面
  • 隨機搜尋（Random Search）：隨機抽樣，高維時比網格更高效
  • 貝氏優化（Bayesian Optimization）：利用歷史結果引導下次搜尋，高效
  • 演化演算法（Evolutionary）：遺傳演算法搜尋，適合複雜空間
🔑 常見超參數
Learning Rate最關鍵，決定收斂速度
Epochs / N_estimators訓練輪數/樹數
Depth / Layers模型複雜度
Dropout Rate正則化強度
Batch Size記憶體需求與收斂穩定性
λ（L1/L2 係數）正則化強度
SLIDE 42L23404公平性

公平性三大指標 + 四階段緩解策略

📊 三大公平性指標
  • 人口均等（Demographic Parity）：各群體正類預測率相等
    P(Ŷ=1|A=0) = P(Ŷ=1|A=1)
  • 機會均等（Equalized Odds）：TPR 和 FPR 在各群體相等
    P(Ŷ=1|Y=1,A=0) = P(Ŷ=1|Y=1,A=1)
  • 補償性公平（ Predictive Parity）：各群體 PPV 相等
    P(Y=1|Ŷ=1,A=0) = P(Y=1|Ŷ=1,A=1)
🛡️ 四階段緩解策略
  • Pre-processing（前處理）：訓練前移除敏感屬性相關性
    → 學習公平表示（Learning Fair Representations）
  • In-processing（訓練中）：約束優化目標加入公平性項
    → Adversarial Debiasing、 fairness constraint
  • Post-processing（訓練後）：調整分類閾值平衡群體
    → 簡單有效，不需重新訓練
  • Data Augmentation：過採樣少數群體/少數類
🤖 RAI 負責任 AI 六原則
  • 公平性：減少偏見與歧視，保障多元群體
  • 可靠性與安全：輸出穩定，防止惡意攻擊
  • 隱私與安全：資料保護與安全存儲
  • 包容性：滿足不同能力群體需求
  • 透明性：演算法可解釋、可追溯
  • 問責：建立責任歸屬與救濟機制
🔑 Explainability vs Interpretability
  • Interpretability（可解釋性）：模型本身可理解
    → 線性迴歸、決策樹、規則提取
  • Explainability（可說明性）：事後解釋黑箱模型
    → SHAP、LIME、Grad-CAM、Counterfactual
SHAP：遊戲理論方法，量化每個特徵的貢獻度
